## DIRECTORY STRUCTURE

### DIRECTORY: .ignore/

### DIRECTORY: doc/
The doc/ directory shelters published content from this project's maintainer(s), e.g., regarding this project's documentation. 

### DIRECTORY: lib/
The lib/ directory shelters content being used somewhere in this directory, as long as the referred content do not belong to the src/ directory. Examples of content are software code (binaries, sources and/or scripts) and PNG files.

### DIRECTORY: sw/
The sw/ directory shelters both binary and text executable files which do not belong to the lib/ directory, e.g., the sw/bin/ subdirectory, which shelters binary executable files, and the sw/src/ subdirectory, which shelters sources, whether executable or not. Also, the install.sh file do not belong to this directory.
